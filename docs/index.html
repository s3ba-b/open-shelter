Open Shelter (OSMP)

An open-source, self-hostable platform that lets animal shelters and rescue organizations run intake, adoptions, fostering, and medical scheduling — without spreadsheets, and without paying for commercial SaaS they can't afford.

View on GitHub See the architecture

Why this exists

Small shelters and rescues run on spreadsheets and ad-hoc tools because commercial shelter-management SaaS is priced for organizations with budgets they don't have. Open Shelter is a free, self-hostable, multi-tenant platform built for the non-commercial sector — and licensed so it can never be captured into a closed, exploitative product.

What it does

Multi-tenancy & isolation

Each shelter is a tenant with data isolation enforced at the data layer (EF Core global query filters over a resolved tenant context) — not just at the UI.

Animal management

Animal records, intake history, and status tracking — available, fostered, adopted, medical hold.

Adoption workflow

Applications, applicant records, and an approval flow from intake to adoption.

Fostering

Foster placements and the people who provide them, tracked end-to-end.

Medical scheduling

Vaccination and treatment schedules with due dates, surfaced through background reminders.

Identity & roles

Token-based auth with tenant claims; role-based access per organization — admin, staff, volunteer.

Built to demonstrate, not just to work

Open Shelter doubles as a reference implementation of cloud-native, multi-tenant architecture on .NET.

.NET 10 .NET Aspire PostgreSQL Redis RabbitMQ OpenTelemetry Docker Compose Azure Container Apps (optional)
  • Aspire-orchestrated services — gateway, identity, business services, and a background worker, with all backing resources wired through the AppHost and visible in the Aspire dashboard.
  • Tenant-aware observability — OpenTelemetry spans tagged by tenant, so tracing stays meaningful in a multi-tenant system.
  • Isolation as a release gate — an automated cross-tenant test suite covers every tenant-scoped entity and endpoint; a failing isolation test blocks a release.
  • Cloud-agnostic by default — primary deployment is Docker Compose, runnable on free/local infrastructure, with Azure Container Apps documented as an optional path.

License: built to stay open

Open Shelter is licensed under AGPL-3.0. Anyone may self-host it, modify it, even charge for running it — but anyone who offers it as a service must publish their complete source, including modifications, under the same license. For a multi-tenant SaaS, that closes off the one move that usually kills open-source projects: fork it, close it, sell it. Any improvement returns to the community by construction.
